## 1️⃣ Answer To The Question No 1:
🎯 Key differences between DOM selection methods

getElementById() → Selects exactly one element using its unique ID attribute.

getElementsByClassName() → Selects a live HTMLCollection of all elements that share a specific class.

querySelector() → Selects the very first element that matches a given CSS selector.

querySelectorAll() → Selects a static NodeList of all elements that match a given CSS selector.

Here is the code-snap to understand their syntax:

```js
const mainTitle = document.getElementById("header-title");

const productCards = document.getElementsByClassName("card");

const firstActiveTab = document.querySelector(".tab.active");

const allExternalLinks = document.querySelectorAll("a[target='_blank']");
```

## 2️⃣ Answer To The Question No 2:
🎯 How to Create and Insert a New Element into the DOM

Step 1: Initialize the element using document.createElement().

Step 2: Inject text, HTML, or classes using properties like .textContent or .classList.

Step 3: Place it inside an existing DOM element using methods like .append() or .appendChild().

Here is the code-snap to understand the syntax:

```js
// 1. Create
const newParagraph = document.createElement("p");

// 2. Add Content
newParagraph.textContent = "This paragraph was generated by JavaScript!";
newParagraph.classList.add("text-highlight");

// 3. Insert
const articleSection = document.getElementById("main-article");
articleSection.append(newParagraph);
```

## 3️⃣ Answer To The Question No 3:
🎯 Event Bubbling and how it works in the DOM

Event Bubbling is an event propagation phase where an event triggers on a specific element and "bubbles" upward.

It travels from the target child element up through its parent, grandparent, and all the way to the root of the document.

Here is the code-snap to understand the syntax:

```js
// Imagine a button inside a section element

document.getElementById("my-button").addEventListener("click", function () {
  console.log("1. The Button was clicked!");
});

document.getElementById("parent-section").addEventListener("click", function () {
  console.log("2. The Section registered a click due to bubbling!");
});

// If you click the button, BOTH logs will fire in the order above.
```

## 4️⃣ Answer To The Question No 4:
🎯 Event Delegation and its usefulness in the DOM

Event Delegation is a pattern where you attach a single event listener to a parent element to manage events for all its children.

It relies on event bubbling to catch the interactions.

Why it's useful: It saves memory (one listener vs. hundreds) and automatically handles new dynamic child elements added later without needing new listeners.

Here is the code-snap to understand the syntax:

```js
// Attaching one listener to the whole table body instead of every row
document.getElementById("table-body").addEventListener("click", function (event) {
  // Check if the clicked target was a table row (TR)
  if (event.target.closest("tr")) {
    console.log("A row was interacted with!");
  }
});
```

## 5️⃣ Answer To The Question No 5:
🎯 Difference Between preventDefault() and stopPropagation()

preventDefault() → Stops the browser's built-in default action for that event (like a form refreshing the page or a link redirecting).

stopPropagation() → Halts the event from bubbling further up the DOM tree, protecting parent elements from triggering their own listeners.

Here is the code-snap to understand the syntax:

```js
// Stopping default browser behavior
document.querySelector("a.disabled-link").addEventListener("click", function (event) {
  event.preventDefault(); // The link will no longer take you to a new page
});

// Stopping event bubbling
document.querySelector(".modal-content").addEventListener("click", function (event) {
  event.stopPropagation(); // Prevents a click inside the modal from closing the modal backdrop
});
```
